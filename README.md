# 项目名称

云南省企业就业失业数据采集系统

# 项目简介

本项目是面向课程作业的前后端一体化系统实现，用于完成企业就业失业数据的采集、审核、汇总分析与演示推送。

- 系统定位：企业就业失业数据采集与管理系统（课程实现版）
- 角色对象：企业端、市级端、省级端
- 核心问题：把“企业填报-市级审核-省级管理/分析/导出-对外演示推送”做成可运行闭环
- 当前仓库范围：包含前端（React）与后端（FastAPI）完整代码、文档与脚本，支持本地启动演示与课堂验收

# 系统功能概览

## 企业端

- 企业信息修改：可维护企业备案信息并保存草稿
- 备案上报：提交备案，等待省级审批
- 数据填报：按调查期填报，支持暂存与上报
- 数据查询：查询企业历史填报记录
- 通知浏览：查看通知、详情、已读状态更新
- 密码修改：修改当前登录密码
- 手机端上报支持：已实现（响应式 Web，企业备案/填报流程可在移动端操作）

## 市级端

- 数据审核：查看本辖区报表，支持查看详情、审核通过、退回
- 通知发布/管理：新增、编辑、删除市级通知
- 密码修改：修改当前登录密码
- 手机端审核支持：已实现（响应式 Web，市级审核流程支持移动端卡片化与抽屉详情）

## 省级端

- 企业备案：企业备案审批（查看、通过、退回）
- 企业查询：可在备案审批页及查询导出页按条件检索企业/报表数据
- 报表管理：省级审核通过、退回、修改、上报部委、删除校验
- 数据汇总：按调查期聚合全省与地市统计
- 数据查询与导出：支持多条件查询与 Excel 导出
- 多维分析：支持地区/企业性质/行业等维度统计分析
- 图表分析：对比分析、趋势分析图表展示
- 取样分析：样本占比与地区分布分析
- 发布通知：省级通知新增、编辑、删除
- 系统管理：调查期、用户、角色、监控、日志管理
- 国家接口演示推送：API Key 配置、预览、推送、记录查询与重试
- 手机端审核支持：已实现（响应式 Web，省级报表管理审核流程支持移动端）

# 需求变更说明

项目已纳入并实现以下需求调整：

1. 1、2、3 月按半月报执行，其他月份按月报执行
2. 企业可在手机端完成上报相关操作（备案与填报）
3. 市级、省级端可在手机端执行审核相关操作
4. 交付周期调整：从第一周 P0 最小闭环迭代到第二周完整审核流转闭环（按 D1-D5 推进）

说明：以上变更已在 docs/change_requests 与周计划文档中体现并落实。

# 安装与启动方式

## 前端启动

```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

默认访问地址：

- http://127.0.0.1:5173

## 后端启动

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

后端地址与接口文档：

- 服务地址：http://127.0.0.1:8000
- Swagger 文档：http://127.0.0.1:8000/docs
- OpenAPI 描述：http://127.0.0.1:8000/openapi.json

## 生产构建

前端构建：

```bash
cd frontend
npm run build
```

前端构建产物目录：frontend/dist

说明：当前仓库以课程演示交付为主，后端未提供独立容器化部署脚本（可按课程要求后续补充）。

# 主要业务流程演示

## 企业流程

登录 -> 备案信息维护/上报 -> 按调查期填报（暂存/上报） -> 数据查询

## 市级流程

登录 -> 审核企业报表 -> 退回/通过 -> 查看/发布通知

## 省级流程

登录 -> 备案审批 -> 报表管理（审核/修改/上报） -> 数据汇总/分析 -> 导出 -> 通知管理 -> 系统管理 -> 国家接口演示

# 手机端适配说明

当前采用响应式 Web 方案，不是原生 App。

已实现的移动端重点适配如下：

- 企业端：备案页面、数据填报页面、工作台与查询页基础响应式
- 市级端：数据审核页（移动端卡片化列表 + 抽屉详情）
- 省级端：报表管理页（移动端卡片化列表 + 抽屉详情）、系统管理页基础响应式
- 全局：移动端抽屉菜单与按钮操作区适配

# 导出与接口说明

## 数据导出

- 支持导出内容：
  - 省级企业备案列表导出
  - 省级数据查询结果导出
- 导出格式：
  - Excel（.xlsx，前端已接入下载）
  - CSV（后端保留接口）

## 国家接口说明

- 当前为演示推送模式，包含：
  - 推送预览
  - 推送执行
  - 推送记录查询
  - 批次日志查询
  - 失败/待重试批次重试
- 支持模拟状态码（如 400/401/500/503）用于课堂演示异常策略
- 结论：用于课程作业展示，非正式政务环境联调

# 常见问题 / 注意事项

1. 前端默认访问地址
   - http://127.0.0.1:5173
2. 后端接口文档地址
   - http://127.0.0.1:8000/docs
3. 本地数据库为空如何初始化
   - 启动后端时会自动建表并写入默认账号与示例调查期。
   - 如需重置演示数据，可删除根目录 app.db 后重启后端。
4. 导出或推送失败应检查什么
   - 是否使用省级账号
   - 查询条件是否返回有效数据
   - 国家推送是否已配置并启用 API Key
   - 推送模拟状态码是否触发了失败分支
5. 移动端展示异常应检查什么
   - 是否在响应式断点下（md 以下）
   - 浏览器缩放比例与缓存
   - 前端是否为最新构建版本
