# 第二周项目计划（迭代目标：审核流转闭环 + 调查期管理）

## 1. 迭代背景
本周在第一周能力基础上继续迭代，目标是完成“企业填报 -> 市级审核 -> 省级审核 -> 省级上报”的完整流转，并实现需求变更：
- 1/2/3 月采用半月报（每月 2 个调查期）
- 4-12 月采用月报（每月 1 个调查期）

## 2. 第二周目标与验收标准

### 2.1 调查期管理（P0）
- 省级可创建/修改调查期
- 支持字段：period_code、period_name、start_time、end_time、period_type、month_no、half_no、status
- 规则校验：
  - 1/2/3 月只能创建 HALF_MONTH 且 half_no=1/2
  - 4-12 月只能创建 MONTH 且 half_no 为空
  - 1/2/3 月每月最多 2 条，4-12 月每月最多 1 条
- 验收：接口可创建、更新、查询当前有效调查期

### 2.2 企业按调查期填报（P0）
- 企业按 period_code 填报
- 每企业每调查期只能 1 条报表
- 支持暂存与上报
- 保留原有人数与减员原因校验
- 验收：查询页可按调查期查看历史状态

### 2.3 市级审核（P0）
- 市级查看本辖区待审核报表
- 支持详情查看、审核通过、退回修改（退回原因必填）
- 审核结果写入状态流转与日志
- 验收：状态从“待市审核”可流转到“待省审核/市退回”

### 2.4 省级报表管理（P0）
- 省级查看“待省审核”报表
- 支持审核通过、退回修改、上报（本地状态流转）
- 验收：状态从“待省审核”可流转到“省审核通过/省退回”，并可变为“省已上报”

### 2.5 操作日志与数据留痕（P0）
- 日志记录：操作人、时间、动作、对象、原因、前后状态
- 报表关键修改保留历史版本快照
- 验收：可查询日志与报表版本列表

## 3. 工作分解（WBS）
1. 数据模型扩展：调查期、按期报表唯一约束、报表版本表
2. 接口契约扩展：调查期管理、按期填报、市省审核接口
3. 业务逻辑改造：时间窗校验、辖区权限校验、状态机流转
4. 日志与留痕：审计日志 + 版本快照
5. 联调测试：闭环冒烟脚本
6. 文档更新：README、运行说明、变更单

## 4. 进度安排（第二周）
- D1：需求澄清与模型设计
- D2：调查期管理 + 企业按期填报
- D3：市级审核流
- D4：省级审核/上报流 + 日志留痕
- D5：联调、修复、文档提交

## 5. 风险与应对
- 风险：旧数据库结构与新模型不兼容
  - 应对：启动阶段自动重建本地 SQLite（开发环境）
- 风险：调查期规则复杂导致脏数据
  - 应对：创建/更新时集中规则校验
- 风险：角色边界不清导致越权
  - 应对：统一 RBAC 与辖区校验
