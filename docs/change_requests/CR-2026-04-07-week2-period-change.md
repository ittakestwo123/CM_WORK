# 变更单：CR-2026-04-07-week2-period-change

## 1. 变更背景
根据最新工作说明书，数据上报周期发生调整：
- 1/2/3 月改为半月报
- 其他月份保持月报

该变更影响调查期管理、企业填报、审核流转与统计口径。

## 2. 变更内容
1. 新增调查期管理能力（省级维护）
2. 报表从按自然月改为按 period_code
3. 每企业每调查期唯一报表约束
4. 增加市级审核通过状态与省级审核/上报状态
5. 增加报表版本留痕（不直接覆盖历史）

## 3. 影响分析
- 数据库：新增 survey_periods、report_versions；employment_reports 结构调整
- 接口：新增/调整调查期、报表、市级审核、省级审核接口
- 权限：市级按 city_code 进行辖区隔离
- 日志：新增 period_create/period_update、report_city_approve、report_province_approve、report_province_submit 等动作

## 4. 实施结果
- 已完成代码实现与联调
- 第二周闭环链路通过冒烟验证
- 输出文档：week2_project_plan.md、week2_run_guide.md

## 5. 回滚策略
开发环境可直接删除本地数据库并重启服务自动重建。
生产环境需通过数据库迁移脚本执行结构变更。

## 6. 最近需求变更说明（已纳入本次提交）
- 变更点：1、2、3 月需要半个月上报一次数据，其他月份仍为一个月上报一次。
- 影响：已在调查期生成逻辑与文档中标注（见本次提交记录）。
