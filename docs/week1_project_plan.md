# 第一周项目计划（迭代 1）

## 1. 迭代目标
- 迭代周期：2026-03-31 至 2026-04-06（5 个工作日）。
- 迭代主题：实现 P0 最小可用闭环（企业备案 + 数据填报）。
- 验收口径：企业可登录并提交备案，省级可审批备案；已备案企业可在调查期内暂存与上报数据，上报后状态为“待市审核”；关键操作可追溯。

## 2. 本周范围（In Scope）
- 登录与身份认证
- 基础 RBAC（企业/市级/省级）
- 企业备案提交
- 省级备案审批（通过/退回）
- 企业数据填报（暂存/上报）
- 填报核心业务校验
- 审计日志

## 3. 非本周范围（Out of Scope）
- 市级审核通过后流转到省级复核的完整闭环
- 报表统计分析与可视化
- 大规模性能压测与高可用部署

## 4. 工作分解（WBS）
1. 架构与基础设施
- 建立 FastAPI + SQLAlchemy 项目骨架
- 配置 SQLite 开发数据库与依赖注入

2. 认证与权限
- 实现登录接口与 JWT 会话
- 按角色返回工作台入口
- 账号状态校验与 RBAC 拦截

3. 备案流程
- 企业备案信息提交与字段格式校验
- 组织机构代码唯一性校验
- 省级审批通过/退回（退回原因必填）

4. 填报流程
- 已备案 + 调查期校验
- 暂存与上报状态机
- 就业减少场景下的必填与层级匹配校验

5. 审计与可追溯
- 关键操作日志落库（操作人/时间/IP/前后状态/原因）

6. 联调与自测
- 典型流程用例验证
- 异常分支验证（越权、状态非法、校验失败）

## 5. 每日计划
- Day1（03-31）：搭建项目骨架、数据库模型、基础配置。
- Day2（04-01）：完成登录鉴权、RBAC、种子账号。
- Day3（04-02）：完成企业备案提交与省级审批接口。
- Day4（04-03）：完成填报暂存/上报与核心校验。
- Day5（04-06）：补充审计日志、联调自测、形成迭代产出。

## 7. 验收标准
- 企业用户登录成功并获取企业工作台。
- 备案提交后状态为“待备案”。
- 省级可审批通过或退回，退回原因必填。
- 仅“已备案”企业可在调查期内填报。
- 填报支持暂存和上报，上报后状态为“待市审核”。
- 就业减少场景下未填必填项时，返回明确错误信息。
- 审计日志可查询关键行为与前后状态。

## 8. 风险与应对
- 需求边界变更：采用变更单记录，并在每次迭代评审中确认范围。
- 业务规则遗漏：通过用例清单覆盖主路径与异常路径。
- 进度偏差：每日站会同步，优先保障 P0 闭环。

## 9. 依赖与协作
- 需求来源：SRS 文档（就业调查系统）。
- 角色协作：企业侧（填报）、省级侧（审批）、开发侧（实现与自测）。
